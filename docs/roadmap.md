# 实施路线图

## 第一阶段：安装脚本

目标：新服务器可以一条命令完成基础环境和 Agent 安装。

交付：

- 安装 Docker / Compose
- 安装 Nginx
- 安装基础安全工具
- 创建 Agent 用户
- 写入 systemd 服务
- 向主控注册节点

验收：

- 服务器重启后 Agent 自动启动。
- 主控能看到节点在线。
- Agent 能上报资源指标。

## 第二阶段：cloud-init

目标：买服务器时直接填初始化脚本，开机自动接入 MOXI 主控。

交付：

- cloud-init 模板
- 一次性注册 token
- 自动设置 hostname
- 自动安装 Agent

验收：

- 新机器创建后无需手动 SSH 即可出现在主控后台。

## 第三阶段：核心 Agent

目标：实现安全可控的运维动作。

交付：

- 心跳
- 指标上报
- Docker 项目状态
- Nginx 状态
- 白名单任务执行
- 审计日志回传

验收：

- 可以部署、重启、健康检查一个测试项目。
- 任意命令执行不可用。
- 敏感字段不会进入日志。

## 第四阶段：自定义镜像

目标：把已配置好的基础系统保存成云镜像。

交付：

- 镜像制作步骤
- 镜像安全清理脚本
- 首次启动注册流程

验收：

- 用自定义镜像创建的新服务器能自动注册。
- 镜像内没有固定密钥和历史日志。

## 第五阶段：主控后台

目标：在 MOXI 管理端或独立后台管理服务器。

交付：

- 服务器列表
- 服务列表
- 部署任务
- 日志查看
- 告警中心
- 备份中心
- 操作审计

验收：

- 主账号可管理全部服务器。
- 代理管理员不能访问底层服务器能力。
- 所有高危操作需要二次确认。
